## ANN to SNN:
• Conversion methodology:

   conversion: ANN->SNN
   method: layer weight transfer approach
   note: all conv and fc layer weight copied from ANN to SNN
   normalization: batch normalization
   activation: ReLU -> spiking neuron models (Leaky Integrate-and-Fire neurons)

• Spike encoding:
rate coding implementation details: The input images were encoded into spike trains using rate coding.  Each pixel intensity was converted into a sequence of spikes over time, where higher pixel values correspond to higher spike rates. At each time step, input spikes are generated probabilistically based on normalized pixel intensities.

input normalization: Input normalization was applied using CIFAR-10 dataset statistics (mean and standard deviation), ensuring consistent input scaling across time steps.

• Temporal inference: number of time steps T, how predictions are aggregated over time

Number of time steps: Time steps T: 25

how predictions are aggregated over time: At each time step, spikes propagate through the network and membrane potentials accumulate over time.  Final predictions are obtained by aggregating outputs across all time steps. The class with the highest accumulated response is selected as the predicted label.


• Metrics: test accuracy, target: within 5-10% of CNN accuracy, average number of time steps to
stable prediction, total spik count per inference, average firing rate

(b) Discuss the following questions:
• Why can SNNs be more energy-efficient despite longer latency?
SNNs can be more energy-efficient because they are event-driven.  Despite the longer latency,
the total number of ops are lower due to sparsity.

• What information is lost during ANN to SNN conversion?
During the conversion, activation values is from continuous values to discrete spikes over time.  
This loses information.  Also information from ReLU and batch normalization are not perfectly 
preserved.

• How does accuracy degrade during conversion vs direct SNN training?
ANN-to-SNN conversion typically results in a drop in accuracy because continuous activations are approximated using spike rates over a limited number of time steps. This approximation introduces quantization and timing errors.

In contrast, directly trained SNNs using surrogate gradients can achieve better accuracy because the network learns spike-based representations during training. However, training SNNs directly is more difficult and computationally expensive due to temporal dynamics and surrogate gradient approximations.


• How does increasing time steps T affect accuracy and latency?
Increasing the number of time steps T generally improves accuracy because the network has more time to accumulate information and refine spike-based representations.

However, increasing T also increases latency, as inference requires sequential processing across time steps. Additionally, computational cost and memory usage increase approximately linearly with T.

• Why are SNNs slower on GPUs despite sparsity?
GPU are optimized for dense, matrices like the ones on CNN.  However, SNN uses sequential time-step processing and sparse, event-driven computation making it unable to take advantage for GPUs.

• Why are SNNs efficient on neuromorphic hardware such as Intel Loihi?
Intel Loihi is designed for event-driven spike computation and can do this on sparse matrices.

• What challenges arise when extending SNNs to object detection?
Object detection require accurate location information but SNN is event driven which means it could lose location information.
